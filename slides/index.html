关键设计决策

为什么双维度评分？
避免「顶刊平庸论文」和「小刊方法突破」被错误排序。分离学术相关性和实践影响力，加权合并。
为什么 yt-dlp 而非 YouTube API？
API 需要 Key + OAuth + 配额。yt-dlp 零配额纯 Python，配合 WARP 解决反爬。
为什么 SQLite 而非 Redis？
Actions runner 无状态，SQLite 单文件通过 git commit 持久化，零维护。
为什么独立播客管线？
播客文字稿 5000+ 字，信息密度和论文不同，需要独立 prompt 保证质量。
